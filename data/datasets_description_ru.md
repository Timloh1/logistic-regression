# Описание наборов банковских данных

Данные содержат информацию о банковских операциях и транзакциях, включая мошеннические действия, для обнаружения финансового мошенничества.

## Обзор

Наборы данных включают:

1. **final_p2p_log.csv.gz**: Журнал переводов (P2P транзакций) между пользователями
2. **final_trans_log.csv.gz**: Журнал общих банковских транзакций (покупки у продавцов, снятие средств в банкоматах и т.д.)

Эти наборы данных используются для обучения и тестирования моделей обнаружения мошенничества.

## 1. Журнал переводов (final_p2p_log.csv.gz)

Этот набор данных содержит записи о денежных переводах между пользователями (P2P транзакции).

### Структура набора данных

| Название столбца | Тип данных | Описание |
|-----------------|------------|----------|
| UserID | Строка/UUID | Уникальный идентификатор клиента, инициировавшего перевод |
| EventTime | Метка времени | Дата и время транзакции |
| RecipientID | Строка/UUID | Уникальный идентификатор получателя |
| Amount | Число с плавающей точкой | Денежная сумма перевода |
| Currency | Строка | Валюта транзакции (например, "RUB") |
| IsFraud | Логическое значение | Флаг, указывающий, является ли транзакция мошеннической (0=легитимная, 1=мошенническая) |

### Назначение

Этот набор данных используется для:
- Анализа паттернов в одноранговых денежных переводах
- Обучения моделей для обнаружения мошеннических P2P-транзакций

## 2. Журнал транзакций (final_trans_log.csv.gz)

Этот набор данных содержит записи об общих банковских транзакциях, таких как покупки у продавцов, оплата счетов и другие банковские операции.

### Структура набора данных

| Название столбца | Тип данных | Описание |
|-----------------|------------|----------|
| UserID | Строка/UUID | Уникальный идентификатор клиента, совершившего транзакцию |
| EventTime | Метка времени | Дата и время транзакции |
| CardID | Строка/Целое число | Идентификатор карты, использованной для транзакции |
| MerchantID | Целое число | Идентификатор продавца/получателя |
| MerchantType | Строка | Категория продавца (например, "Groceries", "Transport", "Financial Institution") |
| Amount | Число с плавающей точкой | Денежная сумма транзакции |
| Currency | Строка | Валюта транзакции (например, "RUB") |
| Country | Строка | Код страны, где произошла транзакция |
| TransactionSuccessful | Логическое значение | Была ли транзакция успешно завершена |

### Назначение

Этот набор данных используется для:
- Анализа паттернов расходов по различным категориям продавцов
- Обнаружения необычных или потенциально мошеннических транзакций
- Изучения того, как поведение по расходам отличается у легитимных пользователей и мошенников
- Обучения моделей машинного обучения для обнаружения мошенничества на основе характеристик транзакций


## Задача
Необходимо построить модель, которая будет предсказывать мошеннические p2p переводы (флаг IsFraud в журнале переводов final_p2p_log.csv.gz).

В процессе решения задачи необходимо самостоятльно разделить данные на обучающую и тестовую выборки.

Для оценки качества модели необходимо использовать метрики:
- ROC-AUC
- Average Precision
А также подобрать порог для, при котором точность будет максимальной, а покрытие будет не менее 70%


## Инженерия признаков

Эти наборы данных используются вместе для комплексного обнаружения мошенничества. Из журналов транзакций извлекаются различные признаки:

### Из журнала транзакций
- Статистика сумм транзакций (количество, среднее, стандартное отклонение, минимум, максимум, сумма)
- Статистика шагов аутентификации
- Частота и пропорции типов продавцов
- Статистика сумм по типам продавцов

### Из журнала P2P
- Паттерны отправки (статистика сумм, уникальные получатели)
- Паттерны получения (статистика сумм, уникальные отправители)

Эти признаки объединяются для создания богатого набора данных, который отражает поведение пользователей с разных сторон, что делает возможным построение эффективных моделей обнаружения мошенничества.
